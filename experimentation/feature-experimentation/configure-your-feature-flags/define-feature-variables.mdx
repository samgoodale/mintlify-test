## Creating feature variables


<Frame>
![feature-variables](https://help.kameleoon.com/assets/images/feature-variables-68c4bb0e9284ee0021d695d2ec0d7190.png)
</Frame>

<Steps>
<Step>
Navigate to the feature flag or create a new one.
</Step>
<Step>
In the left sidebar, click **Set Up** > **Variables** > **Add Variable**.
</Step>
<Step>
Select the variable's **Type**, **Variable Key**, and **Default Value**.
</Step>
<Step>
Click **Save**.
</Step>
</Steps>

<Note>
**Note:**

To retrieve a feature variable from source code, call the `getVariation()` method. Note that if feature variations are used, Kameleoon automatically buckets the user in a variation and returns the value of the variable that has been defined in the variation (calling the `getVariation()` method is not required).
</Note>

## Benefits of feature variables

Feature variables variabilize parts of feature flag code, enabling dynamic assignment of values to variables without coding the values in source code.

Once the feature variable's code is in production, changing the variable does not require redeployment. For example, if a feature is a discount offer for premium customers, but the discount percentage is uncertain, create a **Discount Percentage** variable in the code and update the variable from the Kameleoon app without changing or pushing new code.

Developers and product owners can work together to variabilize parts of a feature that might change, so product updates are not blocked by development and deployments.
